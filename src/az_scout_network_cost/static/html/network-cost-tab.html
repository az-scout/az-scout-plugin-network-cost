Required
Usage Details / Cost Export

The usage details export contains all billed line items including network-related meters. This is required to analyse actual VNet peering cost.

  1. Sign in to the Azure Portal
  2. Open Cost Management + Billing
  3. Select your EA enrollment billing scope
  4. Go to Usage + charges
  5. Select the month and click Download
  6. Choose Usage Details (Version 2) CSV format
  1. Sign in to the Azure Portal
  2. Open Cost Management + Billing
  3. Select your billing profile
  4. Go to Invoices → select an invoice
  5. Click Download usage and charges
  6. Download the Azure usage and charges CSV
Optional
Price Sheet

The price sheet can help understand negotiated prices or meter-level pricing specific to your agreement. Not required for basic analysis.

Available under Cost Management + Billing → Price Sheet for EA or MCA billing accounts.

This mode analyses traffic volumes, not direct billed cost. Uploaded traffic data will be mapped to public Azure pricing to estimate peering cost. Useful for identifying high-volume inter-region traffic.

Required
Traffic Export

Prepare or export a CSV file containing inter-region traffic data. Two formats are supported:

1. Summarised CSV (recommended)

Columns: source_region, target_region, traffic_gb, and optionally direction.

2. Traffic Analytics export

Exported from Azure Traffic Analytics with region-level aggregation.

Raw NSG flow logs may need preprocessing. For V1, prefer a summarised CSV.
Optional
Region Mapping Hints

If your traffic data uses custom region identifiers, you can provide a mapping to standard Azure region names. Not needed if standard armRegionName values are used.
